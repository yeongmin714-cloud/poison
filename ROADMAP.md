## Phase 3: 🌍 세계관 & 월드맵 디자인 (👑 NEW — 4방위 + 황제국)

> 게임 세계의 전체 구조를 설계합니다.
> **4개의 국가(동/서/남/북) × 각 20영지 + 중앙 황제국 = 총 81개 영지**
> **최종 목표: 황제국 탈환**

### 3.1 — 5개 지역 구성 & 난이도 방사형 구조

> **핵심:** 각 방위의 가장자리(외곽)에서 중앙(황제국)으로 갈수록 난이도가 점점 어려워집니다.

```
              [ 북 ]
      ┌─────────────────┐
      │ ⭐  ←   ⭐⭐⭐⭐ │  ← 바깥(쉬움) → 안쪽(어려움)
 [서] │  ┌───────────┐  │ [동]
      │  │  👑 황제국 │  │
      │  │ (최종목표) │  │
      │  └───────────┘  │
      │ ⭐  →   ⭐⭐⭐⭐ │
      └─────────────────┘
              [ 남 ]
```

#### 방사형 난이도 — 4개 링 구조

각 국가의 20영지는 황제국을 중심으로 **4개의 동심원 링(Ring)** 으로 배치:

| 링 | 위치 | 영지 수 | 설명 |
|:--:|:----|:------:|:------|
| 🟢 **Ring 1 (최외곽)** | 국가 가장자리 | 5개 | 병사 적음, 영주 입맛 단순, 초급 병사 |
| 🟡 **Ring 2** | 중간 바깥 | 5개 | 병사 보통, 영주 기본 입맛 |
| 🟠 **Ring 3** | 중간 안쪽 | 5개 | 병사 많음, 영주 까다로운 입맛 |
| 🔴 **Ring 4 (최내곽)** | 황제국 인접 | 5개 | 병사 최다, 영주 매우 까다로움 |

#### 링별 상세 난이도

| 항목 | Ring 1 (최외곽) | Ring 2 | Ring 3 | Ring 4 (황제국 인접) |
|:-----|:--------------:|:------:|:------:|:-------------------:|
| ⭐ **난이도 티어** | ⭐~⭐⭐ | ⭐⭐~⭐⭐⭐ | ⭐⭐⭐~⭐⭐⭐⭐ | ⭐⭐⭐⭐~⭐⭐⭐⭐⭐ |
| 👥 **병사 수** | 3~5명 | 6~10명 | 11~20명 | 21~40명 |
| 📈 **병사 레벨** | Lv.1~10 | Lv.11~20 | Lv.21~30 | Lv.31~40 |
| 🍽️ **영주 입맛** | 아무 음식 OK | 기본 요리 선호 | 고급 요리만 | 왕실급 최고 요리만 |
| 🧪 **영주 지병** | 없거나 가벼움 | 1개 지병 | 1~2개 지병 | 2~3개 복합 지병 |
| 🛡️ **방어력** | 낮음 | 보통 | 높음 | 매우 높음 |
| 💰 **점령 보상** | 작음 | 보통 | 큼 | 매우 큼 |

#### 지역별 링 난이도 가중치

4개 국가는 시작 난이도가 다르므로, 같은 Ring 번호라도 국가에 따라 실제 병사 수/레벨이 다릅니다:

| 국가 | Ring 1 병사 | Ring 2 병사 | Ring 3 병사 | Ring 4 병사 | 특징 |
|:----|:----------:|:----------:|:----------:|:----------:|:------|
| 🏁 **동 (East)** | Lv.1~3 (3명) | Lv.4~8 (5명) | Lv.9~14 (8명) | Lv.15~20 (12명) | 🟢 **초급, 시작 지역** |
| 🏁 **서 (West)** | Lv.3~6 (4명) | Lv.7~12 (7명) | Lv.13~18 (12명) | Lv.19~25 (18명) | 🟡 중상급 |
| 🏁 **남 (South)** | Lv.5~9 (4명) | Lv.10~15 (8명) | Lv.16~22 (15명) | Lv.23~30 (25명) | 🟠 고급 |
| 🏁 **북 (North)** | Lv.8~12 (5명) | Lv.13~20 (10명) | Lv.21~28 (18명) | Lv.29~40 (35명) | 🔴 **최고 난이도** |
| 👑 **황제국** | — | — | — | Lv.41~50 (50명) | **최종 목표** |

┌─────────────────────┐
│  [동] 창병 Lv.7     │ ← 국가 + 직업 + 레벨
│  호감도: ████░░ 60% │ ← 플레이어에 대한 호감도
│  중독도: ██░░░░ 20% │ ← 약물 중독 진행도
│  체력: ████████ 80% │
└─────────────────────┘

[말걸기]  [음식주기]  [약주기]  [돌아가기]
```

> **병사 상호작용은 플레이어가 E키를 눌러 말을 걸 때만** 활성화됩니다. 그냥 지나칠 때는 자동 반응 없음.

#### 5.3.2 — 병사 레벨 시스템

| 항목 | 설명 |
|:-----|:------|
| 레벨 범위 | Lv.1~50 (영지 난이도에 따라 기본 레벨 결정) |
| ⭐ 영지 | Lv.1~10 병사 |
| ⭐⭐⭐ 영지 | Lv.11~25 병사 |
| ⭐⭐⭐⭐⭐ 영지 | Lv.26~40 병사 |
| 👑 황제국 | Lv.41~50 병사 |
| 레벨당 스탯 | HP+10, 공격력+1, 방어력+0.5 |
| 레벨업 조건 | 시간 경과 + 전투 경험 (AI 자동 성장) |

#### 5.3.3 — 병사 호감도 시스템 (국가 기반)

| 상황 | 호감도 변화 | 설명 |
|:-----|:----------:|:------|
| 🏁 **동일 국가 영지 소유** | +10 | 같은 국가 출신 병사는 우호적 |
| 🏁 **적대 국가 영지 소유** | -20 | 해당 국가 영지를 빼앗은 것이 발각 |
| 🏁 **다수의 적대 영지 소유** | -5 × 점령 수 | 빼앗은 영지가 많을수록 모든 적국 병사의 호감도 하락 |
| 🎁 **선물/뇌물** | +5~30 | 음식, 약물, 골드 선물 |
| 💊 **약물 제공** | +15 (중독도↑) | 중독시키면 일시적 호감도 상승 (장기적 위험) |
| 🗡️ **위협** | +20 (일시적) | 강압적 복종, 시간 지나면 -30 보복 |
| ❤️ **호감도 0 미만** | — | 병사가 적대 태도, 배신/공격 가능 |
| ❤️ **호감도 100** | — | 병사가 플레이어에게 충성, 정보 제공 |

### 5.3.3b — 🍽️ 병사에게 음식/약 주기

> 병사 상호작용 메뉴에서 **"음식 주기"** 또는 **"약 주기"** 선택 시, 해당 아이템의 효과가 병사에게 동일하게 적용됩니다.

| 아이템 종류 | 병사 효과 |
|:-----------|:----------|
| ❤️ **치료약/회복약** | 병사 체력 즉시 회복 (HP 회복량 = 아이템 효과 그대로) |
| 🥩 **구운 고기/요리** | 병사 체력 회복 + 포만감 (일시적 능력치 상승) |
| 💊 **독약** | 병사 중독 (호감도 하락, HP 감소, 전투력 저하) |
| 💜 **마약** | 병사 중독도 상승 (일시적 호감도↑, 장기적 위험) |
| 🧪 **해독제** | 병사 중독 해소 (중독도 50% 감소) |
| ⚡ **버프 물약** | 병사 일시적 능력치 상승 (공격력/방어력/속도) |

- [ ] 인벤토리 아이템을 병사에게 직접 사용 가능 (E키 상호작용 → 음식주기/약주기)
- [ ] 효과는 플레이어가 먹었을 때와 동일한 수치로 적용
- [ ] 독약/마약은 몰래 주기 가능 (호감도 조건 무관, 단 발각 시 호감도 폭락)
- [ ] 병사에게 아이템 사용 시 아이템이 인벤토리에서 차감됨

#### 5.3.4 — 병사 약물 중독 시스템

| 중독도 | 효과 | 시각 효과 |
|:-----:|:------|:---------|
| 0~20% | 영향 없음 | 정상 |
| 21~40% | 가벼운 의존, 호감도 소폭 증가 | 눈 흐릿 |
| 41~60% | 중독, 가끔 실수, 정보 누설 위험 | 떨림, 말 더듬 |
| 61~80% | 심각한 중독, 전투력 30% 감소 | 비틀거림, 환각 |
| 81~100% | 완전 중독, 플레이어 명령에 절대 복종 | 쓰러짐, 횡설수설 |
| 100% 초과 | **과다복용 → 사망** | 즉시 사망 |

> **중독 해소:** 시간 경과 시 서서히 감소 (1일=5%). 해독초/해독제 사용 시 즉시 50% 감소.

#### 5.3.5 — 몬스터 레벨 시스템

> 몬스터도 레벨을 가집니다. 레벨은 체력/데미지/드랍률에 영향을 줍니다.

| 항목 | 설명 |
|:-----|:------|
| 기본 레벨 | 몬스터 티어에 따라 결정 (초반=1~5, 중반=6~15, 후반=16~30) |
| 영지 난이도 보정 | ⭐=레벨 유지, ⭐⭐⭐=+3, ⭐⭐⭐⭐⭐=+10 |
| 레벨당 HP 증가 | 초반+5, 중반+10, 후반+20 |
| 레벨당 데미지 증가 | +1 |
| 레벨 표시 | 몬스터 머리 위에 🟢🟡🔴 레벨 숫자 표시 (E키 상호작용 또는 접근 시) |
| 드랍률 보정 | 레벨 10당 희귀 드랍 확률 +5% |

#### 5.3.6 — 병사 상태 체계

- [ ] 병사 상태창: HP, 충성심(호감도), 중독도
- [ ] 역할 지정: 약초꾼, 사냥꾼, 정보원, 일반 병사
- [ ] 사냥꾼 희박한 확률로 사망
- [ ] 정보원 희박한 확률로 처형
- [ ] 머리 위 HUD: 접근 시 레벨/호감도/중독도 표시 (World Space UI)
- [ ] 병사 국가 소속 데이터 (4국가 중 하나, 병사별 고정)
- [ ] 플레이어 영지 점령 현황에 따른 호감도 동적 변화
- [ ] 호감도 0 미만 시 병사가 적대 전환 (플레이어 공격)
- [ ] 호감도 -30 미만 시 **플레이어 발견 즉시 공격** (선공, 대화 불가)
- [ ] 호감도 -50 미만 시 **경보 발령** — 주변 병사/영주에게 플레이어 위치 알림
- [ ] **병사 사망 시 부활:** 가장 가까운 소속 영지에서 **최대체력의 10%** 로 부활

#### 5.3.7 — 🤝 병사 포섭 시스템

> 호감도가 높은 병사는 플레이어의 영지로 데려올 수 있습니다.

| 조건 | 호감도 필요 | 효과 |
|:-----|:----------:|:------|
| 🟢 **포섭 제안** | 70 이상 | 병사가 플레이어 영지로 이적 |
| 🟡 **선물 병행** | 50 이상 + 선물 | 포섭 확률 50%, 실패 시 호감도 -10 |
| 🔴 **강제 포섭** | 0 이상 + 위협 | 포섭 확률 20%, 실패 시 적대 전환 |
| 👑 **충성 병사** | 100 | 자동 포섭, 추가 조건 없음 |

- [ ] E키 상호작용 → "영지로 데려오기" 옵션 (호감도 조건 충족 시)
- [ ] 포섭된 병사는 플레이어 영지에 배치
- [ ] 포섭된 병사도 국가 소속 유지 (타 국가 영지 점령 시 호감도 영향)
- [ ] 포섭된 병사에게 장비 지급 가능 (방독면, 무기, 방어구)
- [ ] 포섭 최대 인원: 영지 레벨 × 5 (Phase 5.1 연동)

#### 5.3.8 — 🛡️ 장비 내구도 시스템

> 방독면, 무기, 방어구 등 모든 장비는 내구도를 가집니다. 내구도 0 → 장비 파괴.

| 장비 종류 | 기본 내구도 | 내구도 감소 조건 | 수리 가능 |
|:---------|:----------:|:---------------|:--------:|
| 🎭 **천 방독면** | 3회 | 사용할 때마다 -1 | ❌ (재제작) |
| 🎭 **가죽 방독면** | 8회 | 사용할 때마다 -1 | ❌ (재제작) |
| 🎭 **강화 방독면** | 15회 | 사용할 때마다 -1 | ✅ (재료 소모) |
| 🎭 **특수 방독면** | 30회 | 사용할 때마다 -1 | ✅ (재료 소모) |
| 🗡️ **기본 무기** | 20회 | 공격 시 -1 (몬스터/전투) | ✅ (모루) |
| 🛡️ **기본 방어구** | 30회 | 피격 시 -1 | ✅ (모루) |
| 🗡️ **고급 무기** | 50회 | 공격 시 -1 | ✅ (고급 재료) |

- [ ] 모든 장비에 `currentDurability / maxDurability` 속성
- [ ] 내구도 0 → 장비 파괴 (인벤토리에서 제거)
- [ ] 장비 UI에 내구도 게이지 표시 (녹색→노랑→빨강)
- [ ] 수리 가능 장비: 크래프트 테이블에서 재료 소모하여 수리
- [ ] 방독면 내구도는 Phase 4.8과 연동

#### 5.3.9 — 👑 병사 명령 시스템 — 4대 임무

> 포섭된 병사에게 **4가지 전문 임무**를 지시할 수 있습니다.
> 병사의 레벨이 높을수록 임무 효율이 증가합니다.

#### 5.3.9.1 — 기본 명령 (즉시 실행)

| 명령 | 설명 |
|:-----|:------|
| 🛡️ **영지 방어** | 해당 영지에 배치, 외부 침입 시 자동 방어 |
| 🏹 **동행** | 플레이어 따라다니며 전투 지원 |
| 🛑 **대기** | 현재 위치에서 정지 |

#### 5.3.9.2 — 🏁 특사 파견 (외교 + 독살)

> 병사를 특사로 보내 인근 영지와 외교하거나, **독이 든 음식을 전달**할 수 있습니다.

| 임무 | 효과 | 필요 조건 | 위험 |
|:-----|:------|:---------|:----:|
| 🎁 **선물 전달** | 대상 영지 호감도 +5~15 (음식/골드 소모) | 특사 Lv.5+, 선물 아이템 | 없음 |
| 🤝 **우호 제안** | 대상 영지 호감도 +20, 일시적 휴전 | 특사 Lv.10+, 호감도 30+ | 실패 시 호감도 -10 |
| 🕊️ **동맵 제안** | 대상 국가와 동맹 (일정 기간) | 특사 Lv.20+, 호감도 60+ | 까다로움 |
| ☠️ **독살 선물** | **음식에 독을 넣어 영주에게 전달** | 특사 Lv.15+, 독든 음식 | **발각 위험 ⚠️** |

**특사 독살 상세:**

| 상황 | 발각 확률 | 결과 |
|:-----|:---------|:------|
| 🟢 **호감도 70+** | 5% | 거의 안 들킴 |
| 🟡 **호감도 40~70** | 20% | 가끔 발각 |
| 🟠 **호감도 10~40** | 50% | 반반 |
| 🔴 **호감도 10 미만** | 80% | **거의 무조건 발각** |
| 💀 **특사 레벨 낮음** (Lv.10↓) | +20% | 레벨 보정 |
| 👑 **특사 레벨 높음** (Lv.30↑) | -15% | 레벨 보정 |

**발각 시 결과:**
- [ ] 호감도 즉시 **0**으로 하락
- [ ] 해당 영주가 **플레이어에게 선전포고** → 즉시 공격
- [ ] 특사 체포/처형 (병사 영구 소실)
- [ ] 해당 국가의 다른 영지 호감도도 **-20** (소문 퍼짐)
- [ ] 특사 레벨이 높을수록 발각 후 **도주 확률** 있음 (20~50%)

- [ ] 영지 메뉴에서 "특사 파견" 선택 → 특사 병사 선택 → 선물/독든음식 선택
- [ ] 발각 확률 UI에 표시 ("발각 위험: 20%")
- [ ] 독든 음식: Phase 4.2 조합 시스템에서 제작 필요
- [ ] 특사는 이동 시간 소요 (영지 간 거리 비례)

#### 5.3.9.3 — 🕵️ 정보원 파견

> 병사를 정보원으로 위장하여 적 영지에 침투시킵니다.

| 임무 | 효과 | 필요 조건 | 소요 시간 |
|:-----|:------|:---------|:--------:|
| 🔍 **영주 정보** | 영주 선호 음식, 지병 파악 | 정보원 Lv.5+ | 1일 |
| 📋 **병력 정보** | 병사 수, 레벨, 배치 파악 | 정보원 Lv.10+ | 2일 |
| 🗺️ **영지 약도** | 영지 내부 구조, 취약점 파악 | 정보원 Lv.15+ | 3일 |
| 💣 **방해 공작** | 병사 중독, 식량 창고 파괴 등 | 정보원 Lv.20+ + 소모품 | 2일 |

- [ ] 정보원 파견 시 해당 영지 정보가 UI에 표시 (미션 완료 후)
- [ ] 정보원 발각 시 체포/처형 (병사 영구 소실)
- [ ] 정보원 Lv.高 → 발각 확률 감소
- [ ] 정보 수집 후 암살/전쟁 전략 수립 가능

#### 5.3.9.4 — 🌿 약초꾼 임무

> 병사를 약초꾼으로 지정하여 플레이어 대신 약초를 채집합니다.

| 임무 | 효과 | 필요 조건 |
|:-----|:------|:---------|
| 🌿 **자동 채집** | 영지 주변 약초 자동 채집 → 인벤토리 저장 | 약초꾼 Lv.1+ |
| 📦 **정기 수확** | 1일마다 약초 3~10개 획득 (레벨/영지 등급 비례) | 약초꾼 Lv.5+ |
| 🔬 **희귀 탐색** | 희귀 약초 발견 확률 +10% | 약초꾼 Lv.15+ |

- [ ] 약초꾼은 전투 불가 (임무 중에는 영지 내에만 존재)
- [ ] 사망 확률: 매우 낮음 (0.5% /일) — 독사 물림 등
- [ ] 채집한 약초는 플레이어 인벤토리에 자동 추가

#### 5.3.9.5 — 🏹 사냥꾼 임무

> 병사를 사냥꾼으로 지정하여 플레이어 대신 몬스터를 사냥합니다.

| 임무 | 효과 | 필요 조건 |
|:-----|:------|:---------|
| 🥩 **자동 사냥** | 영지 주변 몬스터 사냥 → 고기/재료 획득 | 사냥꾼 Lv.3+ |
| 📦 **정기 수확** | 1일마다 고기 2~5개 + 재료 1~3개 획득 | 사냥꾼 Lv.8+ |
| 🏆 **희귀 사냥** | 희귀 몬스터 재료 획득 확률 +15% | 사냥꾼 Lv.20+ |

- [ ] 사냥꾼은 전투 가능 (몬스터와 교전)
- [ ] 사망 확률: 보통 (3% /일) — 강한 몬스터와 조우 시
- [ ] 사냥 대상 몬스터는 영지 난이도에 비례 (쉬운 영지 = 약한 몬스터)
- [ ] 획득 재료는 플레이어 인벤토리에 자동 추가

#### 5.3.9.6 — 🎭 방독면 & 독안개 연동

- [ ] 플레이어가 독약/마약 사용 시 주변에 독안개 발생
- [ ] 동행 중인 병사가 방독면 미착용 → 병사도 중독
- [ ] 병사 중독 게이지 상승 → 전투력 감소 → 사망 위험
- [ ] 병사에게 방독면을 미리 지급/착용 명령 필요
- [ ] 병사 방독면도 내구도 소모됨 (Phase 5.3.8 연동)

#### 5.3.9.7 — ⛏️ 광부 임무

> 병사를 광부로 지정하여 플레이어 대신 자원을 채집합니다.
> 획득한 자원(나무/돌/철광)은 방독면(Phase 4.8) 및 건축 재료로 사용됩니다.

| 임무 | 효과 | 필요 조건 |
|:-----|:------|:---------|
| 🌲 **벌목** | 영지 주변 나무 자동 벌목 → 나무 획득 | 광부 Lv.1+ |
| 🪨 **채석** | 영지 주변 돌 자동 채광 → 돌 획득 | 광부 Lv.3+ |
| ⛏️ **철광 채굴** | 광산에서 철광석 채굴 → 철광석 획득 | 광부 Lv.8+ |
| 🔥 **제련** | 철광석 → 철괴 제련 (크래프트 테이블 필요) | 광부 Lv.10+ + 제련소 |
| 📦 **정기 수확** | 1일마다 재료 3~10개 획득 (레벨/영지 비례) | 광부 Lv.5+ |

- [ ] 광부는 전투 불가 (임무 중에는 영지 주변/광산에만 존재)
- [ ] 사망 확률: 낮음 (1% /일) — 광산 붕괴/야생동물
- [ ] 채집한 자원은 플레이어 인벤토리/영지 창고에 자동 추가
- [ ] 철광석은 별도 제련 과정 필요 (제련소 Phase 5.6.1 연동)
- [ ] 고급 도구(곡괭이/도끼) 장착 시 채집 효율 증가

#### 5.3.10 — ⚔️ RTS 병사 지휘 시스템

> 동행 병사가 있을 때 플레이어의 전투 행동에 병사가 반응하고, 마우스로 직접 지휘할 수 있습니다.

#### 5.3.10.1 — 적군 어그로 (인식 범위)

| 상황 | 반응 |
|:-----|:------|
| 🗡️ **플레이어가 적군 병사 공격** | 해당 전투를 **주변 10m 내 같은 영지 병사가 목격** → 합세하여 플레이어 공격 |
| 👀 **시야 범위** | 적군 병사가 전투를 직접 **보거나 들을 수 있는 거리** (10~15m)에서만 합세 |
| 🏃 **추가 호출** | 합세한 병사가 전투 중 → 주변 10m 추가 병사도 연쇄 합세 (최대 3중) |
| 🛡️ **방어 태세** | 적군 병사가 전투 중 사망 → 남은 병사 경계 태세 강화 (감지 범위 2배) |

#### 5.3.10.2 — 동행 병사 전투 행동

| 플레이어 행동 | 동행 병사 반응 |
|:-------------|:--------------|
| 🚶 **일반 이동/탐색** | 플레이어를 따라다님 (2~3m 간격 유지) |
| 🗡️ **몬스터 공격** (좌클릭) | 동행 병사도 **같은 대상**에게 자동 공격 |
| 🗡️ **적군 병사 공격** (좌클릭) | 동행 병사도 **같은 대상**에게 자동 공격 |
| 🛑 **공격 중단 명령** (H키) | 병사가 즉시 공격 중단, 플레이어에게 귀환 |
| 🏃 **전투 후 이동** | 공격 대상 사망 시 자동으로 플레이어에게 귀환 |

> **전투 중:** 동행 병사는 플레이어를 따라다니지 않고 **전투 위치에 고정**되어 공격만 진행.
> 전투 종료 후 자동으로 플레이어를 따라오기 재개.

#### 5.3.10.3 — 마우스 드래그 선택 & 그룹 명령

> 마우스 드래그로 여러 동행 병사를 한 번에 선택하여 명령을 내릴 수 있습니다.

| 동작 | 설명 |
|:-----|:------|
| 🖱️ **드래그 선택** | 마우스 왼쪽 버튼 누른 상태로 영역 드래그 → 드래그 영역 내 동행 병사 선택 |
| ✅ **선택 표시** | 선택된 병사 머리 위에 파란색 원형 표시 |
| 🖱️ **우클릭 (공격)** | 선택된 병사들이 대상(몬스터/적군)에게 이동하여 공격 |
| 🖱️ **우클릭 (이동)** | 빈 땅 우클릭 → 선택 병사들이 해당 위치로 이동 |
| 🛑 **H키** | 선택된 병사들 공격 중단 + 플레이어에게 귀환 |
| 🔄 **Shift+드래그** | 기존 선택 유지하며 추가 선택 |

#### 5.3.10.4 — 병사 단독 공격 명령

> 플레이어가 직접 공격하지 않아도 동행 병사만 따로 지정하여 공격을 명령할 수 있습니다.

| 명령 방법 | 효과 |
|:----------|:------|
| 🟢 **드래그 선택 + 적 우클릭** | 선택 병사들이 대상에게 이동 후 공격 (플레이어는 공격 안 함) |
| 🔵 **병사 개인 우클릭** (미선택 상태) | 가장 가까운 병사 1명이 대상에게 이동 후 공격 |
| ⭐ **Ctrl+우클릭** | 모든 동행 병사가 대상에게 일제 공격 |

**명령 상태 표시:**

```
[동행 병사: 3명]
  🗡️ 전투 중 → 전리품 곰 Lv.5
  🚶 대기 중
  🗡️ 전투 중 → 적 병사 Lv.3
```

- [ ] 드래그 선택 구현: 마우스 버튼 다운 → 영역 Rect 계산 → 포함된 병사 선택
- [ ] 선택된 병사 시각 피드백 (파란색 원/테두리)
- [ ] 우클릭 명령: 공격/이동 (대상 종류에 따라 자동 판단)
- [ ] 공격 중단 (H키): 선택 병사들의 공격 행동 즉시 중지
- [ ] 병사 전투 AI: 공격 명령 받은 대상에게 이동 → 사거리 도달 시 공격
- [ ] 전투 종료 후 자동 귀환: 모든 대상 사망 or 공격 중단 시 플레이어에게 복귀

#### 7.1.2 — 경보 시스템 (전영지 발령)

> 문지기 병사가 공격을 당하면 **영지 내 모든 병사가 우르르 쏟아져 나옵니다.**

| 단계 | 상황 | 설명 |
|:----:|:-----|:------|
| 1️⃣ | 🗡️ **문지기 공격 시작** | 플레이어가 문지기에게 첫 공격 |
| 2️⃣ | 📯 **경보 울림** | 문지기 중 1명이 생존 시 경적/외침 → 영지 내부에 알림 |
| 3️⃣ | 🏃 **전 병사 출격** | 영지 내부 병사 전원이 입구로 이동 (3~8초 후 도착) |
| 4️⃣ | ⚔️ **총력전** | 모든 병사가 플레이어와 동행 병사에게 일제 공격 |
| 5️⃣ | 🚨 **인접 영지 전파** | 전투가 30초 이상 지속 시 인접한 아군 영지에도 경보 전파 (병사 지원) |

> **은신 상태**로 문지기를 통과하거나 **몰래 침투** 시 경보가 울리지 않음.
> 단, 영지 내부에서 공격/발각 시 그 시점부터 경보 발생.

- [ ] 문지기 병사 2~5명 배치 (영지 입구 오브젝트)
- [ ] 문지기 공격 시 경보 트리거 (영지 내 병사 전원 호출)
- [ ] 경보 후 3~8초 딜레이 (병사가 영지 구조물에서 나오는 시간)
- [ ] 인접 영지 경보 전파 (전투 30초↑)

#### 7.1.3 — 영주 항복 & 처형

> 모든 병사를 처치하면 영주가 **항복 깃발**과 함께 모습을 드러냅니다.

```
전 병사 처치
    ↓
영주 등장 (흰색 항복 깃발 + 손 들고 나옴)
    ↓
┌── 선택지 ──────────────────────┐
│  ⚔️ 처형한다                      │
│     → 영주의 금화 + 병력 전부 획득 │
│     → 영지 아이템/창고 그대로 보존  │
│                                  │
│  🕊️ 살려준다                      │
│     → 영주는 도망감               │
│     → 영지는 획득하나 금화 절반만  │
│     → 해당 국가 호감도 소폭 상승   │
└──────────────────────────────────┘
```

| 선택지 | 효과 | 보상 |
|:------|:------|:------|
| ⚔️ **처형** | 영주 즉시 사망 | 영주의 금화 전부 + 병력 전부 + 창고 아이템 전부 획득 |
| 🕊️ **살려줌** | 영주 도망 (재등장 없음) | 금화 50%만 획득, 병력 절반만 획득, 해당 국가 호감도 +10 |

- [ ] 전 병사 사망 시 영주 등장 애니메이션 (항복 깃발)
- [ ] 선택지 UI: 처형 / 살려주기
- [ ] 처형 선택: 금화+병력+창고 전부 플레이어 소유
- [ ] 살려주기 선택: 절반 보상 + 호감도+10

#### 7.1.4 — 독살 점령

## 🎬 Phase 0.5: 🤖 Animation Rigging 기반 애니메이션 시스템 (신규)

> **사장님이 3D 모델(GLB)을 제공하면, Hermes가 Animation Rigging 패키지로 애니메이션을 직접 제작합니다.**

> 별도 애니메이션 파일(.anim) 없이 코드로 동작을 생성.

### 0.5.1 — Animation Rigging 개요

| 항목 | 설명 |
|:-----|:------|
| 📦 **패키지** | `com.unity.animation.rigging` 1.3.0 |
| 🎯 **원리** | 본(Bone)에 실시간 제약(Constraint)을 걸어 목표물 추적 |
| 🆚 **기존 방식과 차이** | .anim 파일 필요 없음, C# 코드로 즉석 생성 |
| 🧠 **적용 방식** | 서브에이전트가 C# 스크립트로 Rig 레이어 자동 구성 |

### 0.5.2 — 필요한 3D 모델 본(Bone) 구조

> 사장님이 GLB를 제공하실 때, 아래 본이 포함되어 있어야 Animation Rigging이 작동합니다.

**휴머노이드 본 (최소 요구사항):**

```
Hips (루트)
├── Spine → Chest → Neck → Head
├── LeftUpperLeg → LeftLowerLeg → LeftFoot
├── RightUpperLeg → RightLowerLeg → RightFoot
├── LeftUpperArm → LeftLowerArm → LeftHand
└── RightUpperArm → RightLowerArm → RightHand
```

| 본 | 필수 | 용도 |
|:---|:----:|:-----|
| Hips | ✅ | 몸 전체 기준 |
| Spine/Chest/Neck/Head | ✅ | 상체 숙임, 고개 돌림 |
| LeftArm/LeftForeArm/LeftHand | ✅ | 무기/도구 조작, 던지기 |
| RightArm/RightForeArm/RightHand | ✅ | 무기/도구 조작, 던지기 |
| LeftLeg/LeftFoot | ✅ | 걷기, 달리기 |
| RightLeg/RightFoot | ✅ | 걷기, 달리기 |
| Finger bones | ❌ | 선택사항 (손가락까지는 불필요) |

### 0.5.3 — 제작 가능한 애니메이션 목록

| 애니메이션 | 사용 Constraint | 설명 |
|:----------|:---------------|:------|

| 🏃 **Run** (달리기) | Two Bone IK (발) + Override Transform | 빠른 보행, 몸 기울임 |
| 🦘 **Jump** (점프) | Multi-Position (다리) | 점프 포즈 (공중/착지) |
| 🪓 **채집** (Herb pickup) | Two Bone IK (손) | 손이 약초 위치로 이동 |
| 🔨 **제작** (Craft) | Two Bone IK (손) | 양손이 제작대 위치로 이동 |

### 0.5.4 — GLB 투입 후 자동화 파이프라인

```

> ## ⚠️ Hermes 개발 철칙 (필수 준수)
>
> 사장님이 명령을 내리면 **반드시** 아래 순서를 따라야 합니다:
>
> ### 1️⃣ 서브에이전트 활용 (delegate_task)
> - 모든 코딩/QA 작업은 **서브에이전트**에게 위임할 것
> - Hermes 자신은 **디렉터** 역할 (명령/설계/승인)
> - 서브에이전트가 없는 단순 작업(파일 수정 1~2건)만 직접 처리
> - **3개 이상 파일 수정, 새 기능 구현, 리팩토링** → 반드시 delegate_task 사용
>
> ### 2️⃣ TDD — 반드시 컴파일 + 테스트 검증
> - 코드 수정 후 **무조건** Unity batchmode로 컴파일 검증할 것
> - `./run_tests.sh` 또는 직접 Unity -batchmode -executeMethod 실행
> - 컴파일 오류 발생 시 → RED 상태. 수정 후 GREEN 될 때까지 반복
> - **"Scripts have compiler errors" 메시지 없을 때까지** 수정을 멈추지 말 것
> - 테스트 통과까지 완료해야 해당 작업이 ✅ 완료
>
> ### 3️⃣ 오류 수정 프로세스
> ```
> 오류 발견
>   ↓

>   ↓
> Unity batchmode 컴파일 재확인 (직접 실행)
>   ↓
> 새 오류 없음 → 완료
