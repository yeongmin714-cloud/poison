## 1. 🗡️ 복수명부 시스템 (Revenge List)

### 개요
왕(아버지)이 독살당하는 오프닝 컷씬에서, 왕이 남긴 **복수명부**를 발견하는 장면 추가.
플레이어는 복수명부를 아이템으로 소지하며, 이유도 모른 채 영주들을 처단해야 함.

### 상세 설계

**1.1 오프닝 컷씬 확장**
- 기존: 왕 독살 → 흑백 연출
- 추가: 왕이 죽기 직전 복수명부를 플레이어에게 건네는 장면
- 복수명부 ItemData 획득 (고유 아이템, 드랍/판매 불가)

**1.2 복수명부 UI (RevengeListWindow)**
- I 키 인벤토리에서 복수명부 클릭 → 전용 UI 오픈
- 각 영주(10명)의 얼굴(Placeholder 컬러 원형) + 이름 표시
- 복수 이유는 숨겨짐 ("???")
- 영주 처치 시 해당 영주의 복수 이유가 표시됨 ("독약을 제조했다", "독을 운반했다" 등)

**1.3 복수 이유 시스템**
- 각 영주는 `RevengeReason`을 가짐 (Random at game start)
- `PoisonRelated` (10명): 독살과 직/간접적으로 연관
  - 독약 제조자 (2명), 독약 운반자 (2명), 음식 제조자 (2명), 경비/문지기 (2명), 정보 제공자 (1명), 주모자 (1명)
- `NotPoisonRelated` (나머지 영주): 다른 이유 (배신, 약속 위반, 도난 등)
- 영주 사망 시 `RevealReason()` 호출 → 복수명부 업데이트

**1.4 추궁 시스템 (Interrogation)**
- 영주를 영지에서 직접 처형할 때(기존 MercyUI) → **"추궁하기" 버튼 추가**
- 추궁 성공 시 왕의 복수 이유를 직접 들을 수 있음
- 성공 확률: 호감도/레벨/중독 영향
- 추궁 성공 시 해당 영주가 독살 관련자인지 여부 밝혀짐

**1.5 능력치 보상**
- 독살 관련 영주 발견/처치 시 능력치 상승
- 스택 가능: 찾을수록 점점 더 강해짐

### 사이클: 12개

## 2. 🪖 병사 3단계 모델 시스템

### 개요
기존 5단계 레벨 그룹(Novice→Legendary)의 시각적 구분자를
**3개 모델(Lv1-20 / Lv20-40 / Lv40-50)** 로 단순화

### 상세 설계

**2.1 3단계 매핑**
| 레벨 범위 | 등급명 | 접미사 |
|:---------:|:------:|:------:|
| Lv.1 ~ 20 | 초급 (Recruit) | `_tier1` |
| Lv.20 ~ 40 | 중급 (Veteran) | `_tier2` |
| Lv.40 ~ 50 | 고급 (Elite) | `_tier3` |

**2.2 LevelGroupManager 수정**
- 기존 5단계 → 3단계로 변경
- 기존 _tier4, _tier5 → _tier3에 통합 (Lv.40+)

**2.3 ModelSwapper 연동**
- soldier_tier1.glb, soldier_tier2.glb, soldier_tier3.glb 파일 준비
- 없으면 기존 soldier.glb 로 폴백

### 사이클: 3개

## 3. ⏰ 시간/날짜 표시 + 침대 + 저장 시스템

### 개요
게임 시작일로부터 경과 날짜 표시, 침대에서 수면 가능,
수면 시 게임 저장 (슬롯 3개)

### 상세 설계

**3.1 날짜 표시 (HUD)**
- 기존 TimeManager 주야 시스템에 **날짜 카운터** 추가
- "Day X, HH:MM" 형식으로 HUD 표시
- 게임 시작 = Day 1

**3.2 침대 시스템**
- 침대(Bed)와 상호작용 (E 키)
- 수면 옵션: "몇 시간 잘까요?" (2h / 4h / 6h / 8h / 아침까지)
- 수면 중 시간 가속 (Time.timeScale = 60 또는 더 빠르게)
- 수면 종료 조건: 설정한 시간 도달 or 아침 6시

**3.3 저장 시스템 (SaveSystem)**
- 수면 시 저장: 슬롯 3개 중 선택
- 저장 데이터: 플레이어 위치/체력/인벤토리/레벨, 영지 상태, 퀘스트 진행, 시간/날짜
- JSON 직렬화 (Application.persistentDataPath)
- 저장/로드 매니저 (SaveManager.cs)

### 사이클: 8개

## 4. 💾 저장 파일 선택 로드

### 개요
게임 시작 시 저장 파일 목록을 보여주고 선택 로드

### 상세 설계

**4.1 메인 메뉴 (LoadGameMenu)**
- 게임 시작 시: "새 게임" / "불러오기" / "설정"
- 불러오기: 슬롯 3개 표시 (저장된 날짜/시간/레벨 미리보기)
- 빈 슬롯: "비어있음" 표시

**4.2 저장 파일 포맷**
```json
{
  "slotIndex": 0,
  "playerName": "Player",
  "day": 15,
  "time": "14:30",
  "playerLevel": 10,
  "location": "Territory_North_01",
  "timestamp": "2026-06-16T15:30:00",
  "data": { ... compressed save data ... }
}
```

**4.3 저장 데이터 범위**
- PlayerState: 위치, 회전, 체력, 레벨, 경험치, 스탯
- InventoryState: 아이템 목록, 수량
- QuestState: 퀘스트 진행도
- TerritoryState: 영지 소유 상태
- TimeState: 현재 시간, 날짜
- RevengeListState: 복수명부 진행도

### 사이클: 5개

## 5. 🌙 주야별 몬스터 출현

### 개요
낮과 밤에 출현하는 몬스터 풀을 분리.
밤에 더 강한 몬스터가 더 자주 나타나도록 설정.

### 상세 설계

**5.1 몬스터 스폰 조건**
- 낮 (6:00 ~ 18:00): 기본 몬스터 위주 (Rabbit, Wolf, Boar, Deer, Crow, Snake)
- 밤 (18:00 ~ 6:00): 강한 몬스터 위주 (Slime, Golem, Bat, Banshee, Shadow Assassin)
- 저논 (18:00 ~ 20:00, 4:00 ~ 6:00): 혼합

**5.2 스폰 확률**
| 시간대 | 기본 몬스터 | 강한 몬스터 | 보스급 |
|:-----:|:----------:|:----------:|:------:|
| 낮 | 80% | 15% | 5% |
| 저녁 | 50% | 40% | 10% |
| 밤 | 20% | 60% | 20% |

**5.3 MonsterSpawner 확장**
- 기존 MonsterSpawner에 시간대 체크 로직 추가
- TimeManager / DayNightCycle 연동
- 밤에 몬스터 리스폰 속도 증가 (x1.5)

**5.4 시각적 효과**
- 밤에 몬스터 눈이 빛나는 효과 (emissive material)
- 낮/밤에 따라 스폰 파티클 색상 변경

### 사이클: 5개
