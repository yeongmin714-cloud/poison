```
사이클 시작: RED (테스트 실패)
    ↓
구현: GREEN (테스트 통과)
    ↓
리팩토링: REFACTOR (회귀 없음)
    ↓
사이클 완료 ✅ → PROGRESS.md 업데이트
```

### 3️⃣ 서브에이전트 활용
- **3개 이상 파일 수정** → delegate_task로 위임
- **새 기능 구현** → delegate_task로 위임
- **단순 수정 (1~2개 파일)** → 직접 처리
- 서브에이전트 완료 후에도 PROGRESS.md 기록은 직접 확인 후 작성

### 4️⃣ 사장님 추가/수정사항 처리
사장님이 새로운 요청이나 기존 설계 변경을 지시하면 아래 순서로 처리:

```
사장님 요청 수신
    ↓
1. ROADMAP.md에 반영
   └─ 해당 Phase/섹션에 내용 추가 또는 새 Phase 생성
   └─ 설계 스펙과 설명 위주로 기록 (What/Why)
    ↓
2. CYCLE.md에 세분화
   └─ 요청을 1개 이상의 구체적인 사이클로 변환
   └─ 적합한 Phase에 사이클 번호 추가 (예: C8-31, C8-32)
   └─ 구현 단계 위주로 기록 (How)
    ↓
3. PROGRESS.md에 새로운 사이클 상태 반영
   └─ ⬜ 상태로 추가
    ↓
4. 실행 준비 완료
```

- 컴파일 오류 0
